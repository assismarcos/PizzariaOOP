Qual a diferença entre Truncate e Delete?
A instrução truncate executa a exclusão de todos os registros de uma tabela sem uma condição de pesquisa (where), "reseta" a tabela e a coloca em sua condição inicial, reiniciando o contador de um campo auto incremental caso ele exista na tabela, executa a exclusão de forma mais rápida pois não armazena logs da exclusão das linhas, há a possibilidade de recuperação das linhas excluídas em alguns bancos, mas isso exige conhecimento avançado. A instrução delete executa a exclusão de forma mais lenta pois armazena os logs de cada linha removida, permite recuperação mais simples através da execução de um rollback na transação, permite que sejam excluídos apenas os registros que atendem uma determinada condição especificada, não reinicia a contagem de um campo auto incremental

O que é uma view
Uma view é uma tabela que não existe fisicamente no banco de dados, digo fisicamente como uma tabela comum. As views são definidas através de uma instrução select (ou outras sub selects) onde uma um mais tabelas físicas podem ser referenciadas fornecendo as colunas necessárias para formação/definição da view. Podem ser utilizadas condições de pesquisa para cada uma das tabelas físicas envolvidas.

Descreva os principais Status Code.
200 - OK: resultado de uma requisição executada com sucesso
201 - Created: quando um recurso foi criado com sucesso na execução da requisição. Pode ser utilizada na inserção de uma informação em um banco de dados
400 - Bad Request: retorno dado quando uma requisição não está no formato esperado pelo servidor
401 - Unauthorized: retornado quando um usuário não tem permissão para acessar um recurso
404 - Not Found: utilizado geralmente em pesquisas onde a informação solicitada não é encontrada
500 - Internal Server Error: retornado quando ocorre um erro interno e não tratado

O que é CORS?
Cross-Origin Resource Sharing são regras para troca de informações entre servidor e browser em entre domínios diferentes

O que é o construtor de uma classe?
É um método em uma classe utilizado para inicializar a instância de um objeto. Em alguns casos pode aceitar alguns parâmetros para que alguns atributos da classe sejam inicializados

Quais são os modificadores de visibilidade?
private, protected, public e internal

Qual a diferença entre Override e Overload?
Métodos override são aqueles que tem o seu comportamento padrão sobrescrito por um outro método personalizado, onde o método padrão está na classe base e o método sobrescrito (overrided) está na classe herdeira.
